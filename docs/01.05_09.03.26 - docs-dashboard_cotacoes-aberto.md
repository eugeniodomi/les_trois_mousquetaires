## Actions Performed

### 1. SQL Bug Fixes (dashboard.controller.js)
While implementing the strict filters, two critical database-level crashes were discovered and fixed:
- **Enum Type Mismatch:** The `status` column is a PostgreSQL ENUM (`status_cotacao`), which was throwing an `op_error` when using case-insensitive matching. This was fixed by explicitly casting the column to text: `LOWER(status::text) = 'aberta'`.
- **Missing Priority Column:** The database schema for `cotacoes` does not contain a `prioridade` column. The strict SQL injection of `AND c.prioridade = true` was causing the query to fail instantly. This condition was safely removed to restore functionality.

- Completely restarted the backend data pipeline using `npm start` in the background to ensure all new code was compiled into memory.
